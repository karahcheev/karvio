# Release Scope

Release Scope connects products, components, risk, milestones, test plans, and test case coverage. It helps teams decide what must be tested before a release and why.

![Release scope navigation](<../../images/Release Scope Navigation.png>)

## What Release Scope Manages

| Area | Purpose |
| --- | --- |
| Products | Business-level applications, services, or product areas. |
| Components | Technical or functional areas with risk and coverage requirements. |
| Component Coverage | Links test cases to products and components. |
| Milestones | Release, sprint, hotfix, or acceptance cycle tracking. |
| Test Plans | Reusable scopes that can create test runs. |

## Typical Flow

1. Create products for tested applications or services.
2. Add components and risk details.
3. Link test cases to products and components.
4. Create milestones for release cycles.
5. Build test plans manually or from product scope.
6. Create test runs from plans and review readiness.
